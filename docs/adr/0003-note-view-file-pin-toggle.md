# 0003. ノートビューでファイルピンを切り替える

日付: 2026-04-23

ステータス: 採用

## コンテキスト

File view のファイルピンは、ファイル一覧で重要な GTD ノートを上位に置くための状態である。従来は File view の一覧にあるピンボタンからだけ変更でき、現在開いている Markdown ノートから同じ状態を直接変更できなかった。

今回の変更では、File view に参加する Markdown ノートを開いているとき、エディタ右上のビュー操作領域から同じファイルピンを切り替えられる必要がある。除外フォルダ内のノートについては、File view に参加しないという既存の意味を維持する。

## 決定

ファイルピンの判定と切り替えは `NextLevelGtdPlugin` に集約する。File view とノートビューのピン操作は、どちらも `settings.pinnedFileNames` を更新し、同じ `FilePin` の照合規則を使う。

ノートビューの操作は `NoteFilePinToggle` が担当する。アクティブな `MarkdownView` が File view 対象の Markdown ファイルを持つ場合だけ `MarkdownView.addAction` でピン操作を追加し、現在のピン状態に応じてラベル、`aria-pressed`、表示クラスを更新する。

ピン状態が変わったときは、開いている File view に通知して一覧を更新し、アクティブなノートビューの操作も再描画する。

## 代替案

- ノートビュー専用のピン状態を追加する案は採用しない。File view の既存ピン状態と競合し、ユーザーが見るピン状態が分かれるため。
- すべての Markdown ノートにピン操作を表示する案は採用しない。今回のピンは File view の並び順に使う状態であり、除外フォルダのノートを File view 上でどう扱うかが未定義のため。
- File view 側だけに切り替え処理を残す案は採用しない。複数の UI から同じ状態を更新するため、保存と通知を 1 つの経路に寄せる必要がある。

## 影響

- File view 対象の Markdown ノートでは、エディタ右上にファイルピン操作が表示される。
- ノートビューからピン留めしたファイルは、File view の pinned ordering を使うタブでピン留め済みとして扱われる。
- recent タブの並び順、Next Actions のピン、ファイルピンのファイル名照合規則は変更しない。

## 結果

ユーザーは現在開いている GTD ノートから File view のファイルピンを切り替えられる。File view とノートビューは同じ保存状態を参照するため、ピン状態の見え方が分かれない。
