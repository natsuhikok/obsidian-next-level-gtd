# 0005. Next Actions のグループ件数を見出しバッジで示す

日付: 2026-04-24

ステータス: 採用

## コンテキスト

Next Actions view は pinned、dated、default、各 context の順で表示グループを構成している。従来はビュー下部に grouped action count を 1 つだけ表示していたが、この値は表示グループの件数を単純合計したものであり、同じアクションが複数グループに現れる場合はその分だけ重複して数えられていた。

今回の変更では、ユーザーが実際に見ている各グループの規模を見出しの位置で把握できる必要がある。既存の grouping ルールや、1 つのアクションが複数グループに現れる振る舞い自体は維持したい。

## 決定

件数は Next Actions view の各グループ見出しにバッジとして表示する。バッジの値は、そのグループに現在表示されているアクション配列の長さをそのまま使う。

件数の計算は新しい集計モデルを導入せず、既存の `NextActionsQuery.displayGroups` が返す表示結果を唯一の基準にする。これにより pinned、dated、default、context のどのグループでも、ユーザーが実際に見ているメンバーシップと件数が一致する。

ビュー下部の aggregate footer count は廃止する。グループ単位の件数と別の意味を持つ集計値を同時に出すと、同じアクションの重複表示時に解釈がぶれるためである。

## 代替案

- footer の aggregate count を残しつつ各グループ件数も追加する案は採用しない。グループごとの件数と aggregate の意味が異なり、特に重複表示がある場合に混乱を増やすため。
- アクションごとに一意な総数を別計算して見出し件数へ流用する案は採用しない。現在の表示モデルでは同じアクションが複数グループへ正しく現れるため、見出し件数が表示内容とずれるため。
- `NextActionsQuery` に footer 専用の追加集計 API を残す案は採用しない。今回の要件で不要になり、表示結果の単一責任を曖昧にするため。

## 影響

- すべての表示グループ見出しに、そのグループで現在表示中のアクション件数が出る。
- 同じアクションが複数グループに表示される場合、それぞれのグループで 1 件として数えられる。
- ビュー下部の grouped action count は表示されなくなる。

## 結果

ユーザーは Next Actions view を上から下へ読む流れの中で、各カテゴリの大きさをその場で判断できる。件数の意味は常に「その見出しの下に今表示されている項目数」に揃う。
