# 0004. ライブ編集でネクストアクションのピンを切り替える

日付: 2026-04-24

ステータス: 採用

## コンテキスト

Next Actions view のアクションピンは、重要な未完了アクションを専用ビューの pinned グループへ維持する状態である。従来は Next Actions view の一覧からだけ変更でき、ノートをライブ編集している場面では同じアクションのピン状態を確認も更新もできなかった。

今回の変更では、ライブ編集モードの Markdown 行で、既存の Next Actions view と同じ規則で認識される未完了アクションに対して、チェックボックスの近くから直接ピン留めを切り替えられる必要がある。ライブ編集側だけ別の保存状態を持つと、同じアクションの表示が surface ごとに食い違う。

## 決定

アクションピンの判定と切り替えは `NextLevelGtdPlugin` に集約する。Next Actions view とライブ編集のどちらも `settings.pinnedActionPins` を読み書きし、既存の `NextActionPin` の照合規則で同じ状態を共有する。

ライブ編集の表示は `LiveEditActionPinToggle` が担当する。CodeMirror の拡張としてライブプレビュー中の Markdown エディタにウィジェットを追加し、既存の `NextActionCollection` で認識された未完了アクション行だけにインラインのピンボタンを描画する。ボタンはチェックボックスの直前に置き、負のマージンで既定の行揃えの外側へ少し張り出させる。

ピン状態が変わったときは、保存後に開いている Next Actions view を再描画し、登録済みのライブ編集エディタにも再計算を促して表示を同期する。

## 代替案

- ライブ編集専用のアクションピン状態を追加する案は採用しない。Next Actions view と表示が分かれ、どちらが正しい状態か分かりにくくなるため。
- DOM 監視だけで Markdown 行へ直接ボタンを差し込む案は採用しない。ライブ編集の再描画やスクロール更新への追従が不安定になりやすく、エディタ拡張より責務が散るため。
- アクション本文ではなく行番号単位でピンを識別する案は採用しない。既存の保存形式と互換性がなく、今回の変更範囲を超えて pin identity を再定義する必要があるため。

## 影響

- ライブプレビュー中のノートでは、Next Actions view と同じ対象アクションに淡いピン表示が出る。
- ピン留めしたアクションはライブ編集と Next Actions view の両方で同じ pinned 状態として扱われる。
- 同一ファイル名と同一アクション本文で照合する既存の pin identity は維持される。

## 結果

ユーザーはノートを編集している流れの中でアクションピンを切り替えられる。専用ビューとライブ編集は同じ保存状態を参照するため、アクションの pinned 状態が surface ごとに食い違わない。
