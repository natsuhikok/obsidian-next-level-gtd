# 0007. Next Actions に一意な総数を表示する

日付: 2026-04-29

ステータス: 採用

## コンテキスト

Next Actions view は pinned、dated、default、context の各グループに同じ action を複数回表示できる。ADR 0005 では、各グループ見出しの件数を「そのグループ内に表示されている項目数」として扱うことにした。

一方で、ユーザーが残りの action 全体量を把握するには、グループ表示上の重複を除いた総数も必要である。特に pinned かつ dated かつ context を持つ action は複数の見出し件数へ現れるため、見出し件数の合算は残件数として使えない。

## 決定

一意な総数は `NextActionsQuery` の `totalActionCount` として公開する。総数は `displayGroups` と同じ eligibility、つまり blocked ではなく、現在の `DateVisibility` が許可する action だけを対象にする。

重複排除の意味は、表示グループ間で同じ underlying action が複数回現れることを総数では 1 件として扱うことである。別ノートや別行に存在する action は、表示本文が同じでも別々の action として数える。

UI では日付表示切り替えと同じ領域に総数を表示し、既存のグループ見出し件数は維持する。これにより、全体量とグループごとの表示量を別の意味の数として同時に確認できる。

## 代替案

- グループ見出し件数を合算して総数にする案は採用しない。同じ action が複数グループに現れる既存表示と相性が悪く、残件数を過大に示すため。
- action 本文で重複排除する案は採用しない。同じ本文のチェックボックスが別ノートや別行にある場合、それらはユーザーにとって別々の action であるため。
- UI だけで総数を計算する案は採用しない。eligibility は `NextActionsQuery` が所有しており、日付表示モードと blocked 判定に対する解釈を分散させないため。

## 影響

- Next Actions view には、現在の日付表示モードに一致する一意な残 action 数が表示される。
- 近い日付表示では未来の scheduled action は総数に含まれず、すべての日付表示では含まれる。
- 各グループ見出しの件数は従来どおり、そのグループに表示される action 数を示す。
- 表示グループの構成、並び順、action の所属規則は変更しない。

## 結果

ユーザーはグループごとの規模を見出しバッジで確認しつつ、表示中の条件に合う実際の残 action 数を総数として確認できる。総数とグループ件数は別の意味を持つため、重複表示がある場合でも残件数を読み違えにくくなる。
